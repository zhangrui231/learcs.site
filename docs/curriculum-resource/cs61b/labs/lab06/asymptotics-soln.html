{:start="7"} 7. mystery finds the zth largest element in the BST (indexed from 1). It does this by calculating the number of elements in the left subtree, then going to the right or left subtree depending on whether we have too many elements in the left subtree or not enough. Note that numLeft will take proportional time to the number of nodes in the left subtree, so the amount of work in one recursive subcall is Θ(number of nodes in left subtree)\Theta(\text{number of nodes in left subtree})Θ(number of nodes in left subtree).

There are no conditions on the tree, so we must consider when the tree is bushy, left-spindly, and right-spindly. The best-case will be when the root of the tree is exactly the zth largest element, in which case it will take Θ(number of nodes in left subtree)\Theta(\text{number of nodes in left subtree})Θ(number of nodes in left subtree) time (because there is no recursive call).

  • Bushy Tree: There are N2−1\frac{N}{2} - 12N​−1 nodes in the left subtree, so the runtime is Θ(N)\Theta(N)Θ(N) in the best case (when z == N / 2).
  • Spindly Tree (Left): There are N−1N - 1N−1 nodes in the left subtree, so the runtime is Θ(N)\Theta(N)Θ(N) in the best case (when z == N).
  • Spindly Tree (Right): There are 000 nodes in the left subtree, so the runtime is Θ(1)\Theta(1)Θ(1) in the best case (when z == 1).

Hence the best-case runtime is Θ(1)\Theta(1)Θ(1).

The worst case occurs when we have to traverse all the way down to the leaves of the tree.

  • Bushy Tree: For the first recursive call, there are about N2\frac{N}{2}2N​ nodes in the left subtree, then there will be about N4\frac{N}{4}4N​ nodes in the left subtree of the second recursive call, and so on, until we hit a leaf. This means that the worst-case runtime is N2+N4+N8+⋯+1∈Θ(N)\frac{N}{2} + \frac{N}{4} + \frac{N}{8} + \dots + 1 \in \Theta(N)2N​+4N​+8N​+⋯+1∈Θ(N) (when z == 1).
  • Spindly Tree (Left): In this case, each recursive call will have one less node in the left subtree as the parent node. This yields a sum of N+(N−1)+(N−2)+⋯+1∈Θ(N2)N + (N-1) + (N-2) + \dots + 1 \in \Theta(N^2)N+(N−1)+(N−2)+⋯+1∈Θ(N2) (when z == 1).
  • Spindly Tree (Right): There are 000 nodes in the left subtree at each recursive call, and the height of the tree is NNN, so the worst-case runtime is Θ(N)\Theta(N)Θ(N) (when z == N).

From the above cases, we see that the worst-case runtime is Θ(N2)\Theta(N^2)Θ(N2).

Summary: Best-case: Θ(1)\Theta(1)Θ(1). Worst-case: Θ(N2)\Theta(N^2)Θ(N2).
